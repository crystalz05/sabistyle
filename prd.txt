1.1 Product Vision
SabiStyle is a Nigerian fashion e-commerce app that lets customers browse clothing, add items to cart, checkout via Paystack, and track their orders — all from a clean, fast mobile experience. The app is built for the African fashion market and is scoped as a production-ready MVP.
1.2 User Types
Customer — browses, wishlists, buys, tracks orders
Admin — manages products, orders, and promo codes via Supabase dashboard (no separate admin app in MVP)

5. How Admin Adds Items to the Store
In MVP, there is no Flutter admin panel. The admin manages all store content directly through the Supabase Dashboard. Here is the exact workflow:
5.1 Adding a Product
Log in to Supabase dashboard at app.supabase.com
Go to Storage → Create a bucket called 'products' (set to public)
Upload product images to Storage → copy the public URLs
Go to Table Editor → open the 'products' table
Click Insert Row and fill in: name, description, price, category_id, images (paste URLs as array), sizes, colors, stock_qty
Set is_featured = true if it should appear on the home feed
Set is_active = true to make it visible in the app
5.2 Adding a Category
Upload category icon to Supabase Storage → copy URL
Insert row in 'categories' table: name, icon_url, display_order
5.3 Managing Orders
All orders appear in the 'orders' table in real time
To update order status: find the order row, edit the 'status' column (pending → processing → shipped → delivered)
Customer sees the status update immediately in the app via Supabase realtime
5.4 Creating Promo Codes
Insert row in 'promo_codes' table: code, discount_type (percentage or fixed), discount_value, max_uses, expires_at
Customer enters the code at checkout — the app validates against this table
Post-MVP plan: Build a simple Flutter admin panel with product upload, image picker, and order management to replace the Supabase dashboard workflow.

8. Payment Flow — Paystack
8.1 Flow
Customer taps 'Pay' on the Checkout Review screen
App calls Paystack initialize endpoint with amount, email, and a unique reference
Paystack popup opens (card / bank transfer / USSD options)
On payment success, Paystack returns the reference to the app callback
App calls Paystack verify endpoint with the reference to confirm payment server-side
On verified: order record is created in Supabase with status = 'processing'
Customer is routed to Order Confirmation screen
8.2 Package
Use flutter_paystack package. Initialize with your public key in main.dart. Use the charge() method with the amount in kobo (multiply Naira × 100).

9.2 Auth Guard
GoRouter redirect checks AuthBloc state on every navigation. If AuthUnauthenticated and route requires auth → redirect to /login. If AuthAuthenticated and route is /login or /signup → redirect to /home.
